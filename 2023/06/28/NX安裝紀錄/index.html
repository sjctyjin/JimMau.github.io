NX安裝紀錄

在Xavier NX安裝lxd系統並建置ORB-SLAM3 ROS系統時遇到的問題統整。

狀態描述:

剛開箱的全新Xavier NX 是以eMMC 16G為系統的儲存磁區，附上一條128G M.2 SSD

預設OS系統為Jetpack 4.6 版本 Base on Ubuntu 18.04，Linux kernel 4.9。

問題1. lxd init 時無法使用zfs

安裝lxd時發現無法使用zfs格式作為lxd storage pool的儲存格式，後發現zfs格式Xavier NX的核心架構arm不支援zfs格式。

原因 :

在筆電上使用VMWare建立的Ubuntu 20.04其核心是x86，因此在建立lxd init時預設是使用zfs格式，但此格式不影響lxc容器使用。

問題 2. lxc容器在Jetpack 4.6無法連接外網

建置 lxc ubuntu20.04容器後，發現無法連線外部網路，使用ping 時會出現 ping: socket: Operation not permitted 無法解決，

後斷定應該是該OS版本與lxc的支援相容性問題，因在ubuntu 20.04版本中不會有這個問題。

且eMMC僅有16GB，無法安裝太大的容量，因此須將OS系統安裝於M.2 SSD上。

問題 3. 將OS系統燒錄至SSD

將SSD安裝上Xavier NX後，需進行磁碟分割，規劃磁區後，將Xavier NX透過micro USB 連接電腦後，使用Nvidia SDK Manager 進行版本移植

把OS系統燒錄至SSD上並作為開機執行檔。

且eMMC僅有16GB，無法安裝太大的容量，因此須將OS系統安裝於M.2 SSD上。

燒錄步驟 來源 : https://blog.cavedu.com/2022/04/20/jetson-xavier-nx-system-setup-2022/

Step1. 先用VMware開啟一個ubuntu18.04的虛擬主機，並在虛擬主機中安裝NVIDIA SDK Manager下載點：https://developer.nvidia.com/nvidia-sdk-manager

Step2.開啟NVIDIA SDK Manager並且將電腦的USB線連接到Jetson Xavier NX的Micro USB中

如有下載過相關JetPack可以取消勾選Host Machine，並且將Target Hardware選擇Jetson Xavier NX modules，TARGET OPERATING SYSTEM需要選擇JetPack4.6以上才可以將系統安裝至SSD當中(本次安裝系統版本為Jetpack 5.1.1 (rev.1))。

Step3. 選擇要下載的項目，本文將使用預設設定來全部下載安裝。請記得 勾選下方的 accept license，如果要馬上安裝，則不要勾選 Download now. Install later。如果想先下載之後再安裝，則可勾選該選項。

Step 4. 開始安裝後輸入密碼後會跳出以下選項。(PAUSE後重新啟動會跳出此選項)

可以選擇自動安裝（如無法自動安裝再選擇手動安裝），IP使用預設IP，帳號密碼請輸入當初建立的帳號及密碼。

最重要的一點是把 5. Storage Device（儲存裝置） 由下拉式選單中選擇 NVMe，也就是 SSD儲存空間，便可以將系統預設安裝至SSD當中。

帳號密碼以及儲存裝置相關設定都正確的話，就可以將系統正確安裝至SSD中。

Step 5. 出現以下畫面代表系統安裝成功。下一段要說明如何透過映像檔燒錄系統到 SSD 來開機

如果是要重新安裝並且自動安裝失敗時可以進入Recovery Mode進行手動安裝安裝。
